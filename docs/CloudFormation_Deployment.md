### Pre-Deployment

Create an S3 bucket, name of your choosing

File structure inside bucket should be two folders "frontend" and "backend".

Inside frontend, include a zipped file of the frontend code (I just asked claude to do this). It should be named frontend-app.zip.

Same for backend. Name backend-app.zip.

For reference, the claude commands for generating the zips

Frontend (run from the project root):
```
npm install
npm run build
cd dist
zip -r ../frontend-app.zip .
cd ..
```

Backend:
```c
d backend
zip -r ../backend-app.zip . \
  --exclude "node_modules/*" \
  --exclude ".vercelignore" \
  --exclude "migrations/*" \
  --exclude "models/*" \
  --exclude "scripts/*" \
  --exclude "seed.js" \
  --exclude "README.md"
cd ..
```

### Deployment

Project should be deployed on CloudFormation using the architecture.yaml template. The parameters should be the following:

Stack Name: commie-prod

AtrifactsBucketName: Whatever you named the S3

KeyPairName: vockey

JwtSecret: random character generated by the CLI prompt: node -e "console.log(require('crypto').randomBytes(32).toString('hex'))".

All others should be filled in by default.

### Re-deployment

In the event of needing to redeploy, ensure that you delete all of the dynamoDB tables before deploying again to avoid errors.
